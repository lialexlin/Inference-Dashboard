# Inference Stack Dashboard — orientation

A personal investing dashboard mapping the AI inference scaling stack across 15 layers. Lets the user spot which layers are still under-priced after the memory run-up. Two coupled views: a **stack overview** (15 layer cards with status and player chips) and a **live signals feed** (earnings/IR/filings/RSS commentary, layer-tagged).

## Architecture

Static HTML + JSON files + Python data layer. **No build step, no framework.** Browser fetches `data/*.json` directly; Tailwind/Alpine/ECharts/marked all via CDN.

```
sources/* → SQLite cache → transform.py (tag) → data/*.json (atomic) → static HTML
```

This was a deliberate choice: portable to GitHub Pages without changes, no toolchain to maintain, easy for the user to evolve.

## How to run

Project is uv-managed. Deps live in `pyproject.toml`, locked in `uv.lock`, materialized into `.venv/` (gitignored). First time on a new machine: `uv sync`.

```bash
uv run python -m jobs.seed              # one-time, after editing jobs/seed.py
uv run python -m jobs.refresh           # daily — prices + RSS + EDGAR
uv run python -m http.server 8000       # serve
uv add <pkg>                            # add a dep (updates pyproject + lockfile + venv)
```

Or **double-click `launch.command`** for one-click open. **`refresh.command`** for one-click data update. Both wrap `uv run`.

## Refresh model

- Curated signals (from `jobs/seed.py:SIGNALS`) are durable — preserved across refreshes via id-prefix check (`rss-*`, `edgar-*` are discovered; rest are curated).
- Partial refreshes don't wipe data: `refresh.py` reads previous `meta.json` and `signals.json` and merges. Skipping RSS doesn't drop existing RSS signals. `--skip-ciq` (or a CIQ stage failure) preserves prior `ciq.*` blocks under `fundamentals.json`.
- Atomic JSON writes (write `.json.tmp` then rename) — frontend never sees half-written state.

## Data source split

| Field | Source | Why |
|---|---|---|
| Daily prices, 1m/6m/YTD changes | yfinance | CIQ has prices but not daily-historical-friendly at scale. yfinance is fast and works for the chart. |
| **Forward P/E, target prices, analyst counts** | yfinance (`Ticker.info`) | Axiom's CIQ Xpressfeed subscription doesn't include forward estimates. |
| **Trailing P/E, EV/EBITDA, P/S, P/B (current + 8y history)** | S&P Capital IQ | Currency-aware, audit-grade, available for all global tickers. yfinance has only current snapshot. |
| **Margins, growth (YoY/3Y/5Y), returns (ROE/ROA/ROIC)** | S&P Capital IQ | Pre-computed and globally consistent. |
| Revenue/EBITDA/NI in USD | S&P Capital IQ | Reporting currency derived from PLAYERS `country` (NOT ticker suffix — TSM and ASML are ADRs that report in TWD/EUR). |
| US filings | EDGAR | — |
| Industry RSS | feedparser | — |

CIQ creds in `.env` (SF_ACCOUNT/SF_USER/SF_PASS — Axiom Asia subscription). Schema reference: `../Exponential-Growth/.claude/skills/s&p/SKILL.md`. Don't commit `.env`.

## How to add things

**Add a layer:** edit `LAYERS` and `LAYER_CONTENT` in `jobs/seed.py`, plus `LAYER_KEYWORDS` in `jobs/transform.py`. Re-run `python -m jobs.seed`.

**Add a player ticker:**
1. Add to `PLAYERS` in `jobs/seed.py` (set `layer_id`, `country` — used to derive CIQ reporting currency, optional `secondary_layers`).
2. If US-listed, add `ticker → CIK` to `CIK_MAP` in `jobs/sources/edgar.py` (look up at sec.gov).
3. Run `python -m jobs.seed --refresh-ciq-mapping` to resolve the new ticker against S&P Capital IQ. If it fails, add `{ticker: companyId}` (or `None` to skip) to `CIQ_OVERRIDES` in `seed.py` — find the right `companyId` via `find_company` in the CIQ skill (see Sources below).
4. Run `python -m jobs.refresh`.

**Add a curated signal:** edit `SIGNALS` in `jobs/seed.py`. Each entry needs a unique `id` that does NOT start with `rss-` or `edgar-` (that prefix is reserved for discovered entries).

**Add an RSS source:** add a tuple to `FEEDS` in `jobs/sources/rss.py` (name, url, source_type ∈ {industry, news, analyst}).

## Known gotchas

- **Browser cache** during dev: when editing JS or CSS, append `?v=N` to the URL or hard-reload. The static server doesn't set cache-busting headers.
- **Nested anchor tags** — chips inside layer cards must be `<span>`, not `<a>`. Browsers hoist nested `<a>` out of the parent, breaking layout. (Hit this once; fixed.)
- **Hidden ECharts containers** — call `chart.resize()` after un-hiding, or the canvas stays 0×0.
- **pyenv blake2 noise** — harmless `ERROR:root:code for hash blake2b was not found` prints on every Python startup. Filterable with `grep -v blake2`. Not a real error.
- **Broken RSS feeds** — The Register, Mule Capital, AnandTech all return malformed XML. Tolerated by feedparser; surfaced on `about.html` as red "error" rows. Not fixable from our side.
- **6967.T (Shinko Electric)** — yfinance returns no data because Shinko is being taken private by JIC. Documented; the chip just renders without a price.
- **CN A-shares (300308.SZ, 300502.SZ)** — yfinance handles these but data can lag; verify on the layer page.
