# Inference Stack Dashboard ‚Äî orientation

A personal investing dashboard mapping the AI inference scaling stack across 15 layers. Lets the user spot which layers are still under-priced after the memory run-up. Two coupled views: a **stack overview** (15 layer cards with status and player chips) and a **live signals feed** (earnings/IR/filings/RSS commentary, layer-tagged).

## Architecture

Static HTML + JSON files + Python data layer. **No build step, no framework.** Browser fetches `data/*.json` directly; Tailwind/Alpine/ECharts/marked all via CDN.

```
sources/* ‚Üí SQLite cache ‚Üí transform.py (tag) ‚Üí data/*.json (atomic) ‚Üí static HTML
```

This was a deliberate choice: portable to GitHub Pages without changes, no toolchain to maintain, easy for the user to evolve.

## How to run

Project is uv-managed. Deps live in `pyproject.toml`, locked in `uv.lock`, materialized into `.venv/` (gitignored). First time on a new machine: `uv sync`.

```bash
uv run python -m jobs.seed              # one-time, after editing jobs/seed.py
uv run python -m jobs.refresh           # daily ‚Äî prices + RSS + EDGAR
uv run python -m http.server 8000       # serve
uv add <pkg>                            # add a dep (updates pyproject + lockfile + venv)
```

Or **double-click `launch.command`** for one-click open. **`refresh.command`** for one-click data update. Both wrap `uv run`.

## Cloud refresh (GitHub Actions)

`.github/workflows/refresh.yml` runs daily at **12:00 UTC** on a GitHub-hosted Ubuntu runner and on manual `workflow_dispatch`. Same `python -m jobs.refresh` as local ‚Äî runs all four stages (CIQ ‚Üí prices ‚Üí RSS ‚Üí EDGAR), atomic writes, then `git add data/ && git commit && git push` as `inference-dashboard-bot`. GitHub Pages auto-redeploys on the push, so the live site refreshes ~2 min after the cron fires.

CIQ stage needs Snowflake creds. Set as GitHub Actions repo secrets (Settings ‚Üí Secrets and variables ‚Üí Actions): `SF_ACCOUNT`, `SF_USER`, `SF_PASS`. Without them the CIQ stage logs `CIQ connect failed`, preserves prior `ciq.*` blocks, and continues with the other stages.

**Race-with-bot gotcha:** if you push from your laptop after the bot has pushed the same day, `git push` rejects. `git pull --rebase` will conflict on `data/meta.json` / `data/prices.json`. Resolve with `git checkout --theirs data/meta.json data/prices.json` (your local data is normally the superset, especially if you ran CIQ locally), then `git rebase --continue`.

## Refresh model

- Curated signals (from `jobs/seed.py:SIGNALS`) are durable ‚Äî preserved across refreshes via id-prefix check (`rss-*`, `edgar-*` are discovered; rest are curated).
- Partial refreshes don't wipe data: `refresh.py` reads previous `meta.json` and `signals.json` and merges. Skipping RSS doesn't drop existing RSS signals. `--skip-ciq` (or a CIQ stage failure) preserves prior `ciq.*` blocks under `fundamentals.json`.
- Atomic JSON writes (write `.json.tmp` then rename) ‚Äî frontend never sees half-written state.

## Data source split

| Field | Source | Why |
|---|---|---|
| Daily prices, 1m/6m/YTD changes | yfinance | CIQ has prices but not daily-historical-friendly at scale. yfinance is fast and works for the chart. |
| **Forward P/E, target prices, analyst counts** | yfinance (`Ticker.info`) | Axiom's CIQ Xpressfeed subscription doesn't include forward estimates. |
| **Next earnings date** | yfinance (`Ticker.calendar`, `Ticker.earnings_dates` fallback) | Stored at `fundamentals[ticker].next_earnings = {date, range_end, confirmed}` (or `null`). | **Trailing P/E, EV/EBITDA, P/S, P/B (current + 8y history)** | S&P Capital IQ | Currency-aware, audit-grade, available for all global tickers. yfinance has only current snapshot. |
| **Margins, growth (YoY/3Y/5Y), returns (ROE/ROA/ROIC)** | S&P Capital IQ | Pre-computed and globally consistent. |
| Revenue/EBITDA/NI in USD | S&P Capital IQ | Reporting currency derived from PLAYERS `country` (NOT ticker suffix ‚Äî TSM and ASML are ADRs that report in TWD/EUR). |
| US filings | EDGAR | ‚Äî |
| Industry RSS | feedparser | ‚Äî |

CIQ creds in `.env` (SF_ACCOUNT/SF_USER/SF_PASS ‚Äî Axiom Asia subscription). Schema reference: `../Exponential-Growth/.claude/skills/s&p/SKILL.md`. Don't commit `.env`.

## How to add things

**Add a layer:** edit `LAYERS` and `LAYER_CONTENT` in `jobs/seed.py`, plus `LAYER_KEYWORDS` in `jobs/transform.py`. Re-run `python -m jobs.seed`.

**Add a player ticker:**
1. Add to `PLAYERS` in `jobs/seed.py` (set `layer_id`, `country` ‚Äî used to derive CIQ reporting currency, optional `secondary_layers`).
2. If US-listed, add `ticker ‚Üí CIK` to `CIK_MAP` in `jobs/sources/edgar.py` (look up at sec.gov).
3. Run `python -m jobs.seed --refresh-ciq-mapping` to resolve the new ticker against S&P Capital IQ. If it fails, add `{ticker: companyId}` (or `None` to skip) to `CIQ_OVERRIDES` in `seed.py` ‚Äî find the right `companyId` via `find_company` in the CIQ skill (see Sources below).
4. Run `python -m jobs.refresh`.

**Add a curated signal:** edit `SIGNALS` in `jobs/seed.py`. Each entry needs a unique `id` that does NOT start with `rss-` or `edgar-` (that prefix is reserved for discovered entries).

**Add an RSS source:** add a tuple to `FEEDS` in `jobs/sources/rss.py` (name, url, source_type ‚àà {industry, news, analyst}).

## Scoring (Business √ó Valuation ‚Üí quadrant)

Each ticker gets a `business` and `valuation` z-score (positive = good), composed from components z-scored within the ticker's primary-layer peer set. Quadrants:

| business | valuation | label |
|---|---|---|
| > 0.3 | > 0.3 | **underappreciated** (top idea ‚Äî strong growth, cheap) |
| > 0.3 | < ‚àí0.3 | **priced-in** (strong growth but already rallied) |
| < ‚àí0.3 | > 0.3 | **value-trap** (cheap but no growth) |
| < ‚àí0.3 | < ‚àí0.3 | **avoid** |
| else | else | **fair** |

Layer status rolls up the player quadrants:
- ‚â•30% underappreciated ‚Üí underappreciated
- ‚â•50% priced-in ‚Üí priced-in
- distribution spread ‚Üí mixed
- else ‚Üí fairly-priced

`status_override` in seed.py LAYERS pins the label when set. Displayed `status` = `override or auto`.

### `data/manual_estimates.json` shape (per ticker, all fields optional):

```json
{
  "NVDA": {
    "as_of": "2026-05-12",
    "fwd_eps_curr": 6.15,
    "fwd_eps_history": [
      {"as_of": "2026-02-15", "value": 5.40}
    ],
    "fwd_pe_5y_median": 38.0,
    "guidance_note": "Backlog +25% qoq; capex raised"
  }
}
```

Minimum to power EPS revision scoring: `fwd_eps_curr` + at least one prior snapshot. The user updates these post-earnings or monthly from Capital IQ / Bloomberg. History accumulates as entries are added. Coverage is surfaced as a row on `about.html`.

Threshold tuning lives in `THRESH`, `ROLL_UNDER_FRACTION`, `ROLL_PRICED_FRACTION` constants at the top of `jobs/scoring.py`.

## Known gotchas

- **Browser cache** during dev: when editing JS or CSS, append `?v=N` to the URL or hard-reload. The static server doesn't set cache-busting headers.
- **Nested anchor tags** ‚Äî chips inside layer cards must be `<span>`, not `<a>`. Browsers hoist nested `<a>` out of the parent, breaking layout. (Hit this once; fixed.)
- **Hidden ECharts containers** ‚Äî call `chart.resize()` after un-hiding, or the canvas stays 0√ó0.
- **pyenv blake2 noise** ‚Äî harmless `ERROR:root:code for hash blake2b was not found` prints on every Python startup. Filterable with `grep -v blake2`. Not a real error.
- **Broken RSS feeds** ‚Äî The Register, Mule Capital, AnandTech all return malformed XML. Tolerated by feedparser; surfaced on `about.html` as red "error" rows. Not fixable from our side.
- **6967.T (Shinko Electric)** ‚Äî yfinance returns no data because Shinko is being taken private by JIC. Documented; the chip just renders without a price.
- **CN A-shares (300308.SZ, 300502.SZ)** ‚Äî yfinance handles these but data can lag; verify on the layer page.

## Framework view ‚Äî Demand √ó Bottleneck √ó Moat √ó Exit

The dashboard supports a Druckenmiller-style bottleneck-rotation thesis on inference scaling. Five primitives operationalize the framework:

| Primitive | Where | Data file | Stage |
|---|---|---|---|
| **Demand telemetry** (variant perception) | Strip at top of `index.html` | `data/demand_signals.json` | `jobs/sources/openrouter.py` |
| **Bottleneck heat-map** (which layer binds now) | Heat meter on each layer card; callout on `layer.html` | `data/bottleneck.json` | `jobs/bottleneck.py` |
| **Choke-point moat** | Existing Business √ó Valuation score | `data/scores.json` | `jobs/scoring.py` |
| **What's priced in** (reverse-DCF) | Column on `stocks.html`; card on `stock.html` | Inside `data/scores.json` (`implied_5y_cagr`, `implied_vs_recent_growth`) | `jobs/scoring.py::_implied_cagr` (r=10%, terminal P/E=18, bisection) |
| **Exit triggers** | 4-tile panel on `index.html` | `data/exit_triggers.json` | `jobs/exit_triggers.py` |
| **Architectural risk** (thesis-killer feed) | Strip on `index.html`, filter chip on `signals.html` | `arch_risk: bool` on every signal | `jobs/transform.py::ARCH_RISK_KEYWORDS` |

Stage order in `jobs/refresh.py`: `CIQ ‚Üí prices ‚Üí RSS ‚Üí EDGAR ‚Üí transform/tag ‚Üí bottleneck ‚Üí openrouter ‚Üí scoring ‚Üí exit_triggers`. All atomic writes; every stage preserves prior output on failure.
